Tabla de decisión — Ecosistema de diseño VMS Energy

Guía canónica para decidir qué concepto de diseño aplicar a cada página nueva o migrada. Actualizar este documento antes de crear cualquier plantilla nueva.

v1.0 · Mayo 2026 · Complemento de tokens.css
Cómo aplicar: Agrega el atributo data-concept directamente en la etiqueta <html> de cada página. Para páginas sectoriales, agrega también data-sector. Luego vincula tokens.css antes de cualquier otro CSS.
Ejemplo: <html lang="es-MX" data-concept="sectorial" data-sector="mineria">

Árbol de decisión — ¿qué concepto usar en una página nueva?

¿La página vende o presenta un producto físico (variador, tablero, equipo) para cotizar?
SÍ → ¿Es una ficha de producto con precio/datasheet? → E-shop · Si es LP de venta directa → Equipamiento
NO ↓
¿La página describe un servicio técnico muy especializado (turbogeneradores, mantenimiento, oil & gas, sistemas de supresión)?
SÍ → Servicios Especializados · índigo · landing page
NO ↓
¿La página está orientada a un sector industrial específico (minería, agroindustria, alimentos)?
SÍ → Sectorial + sector correspondiente: mineria / agroindustria / alimentos-bebidas
NO ↓
¿La página presenta capacidades de la empresa, equipo, procesos o es punto de contacto?
SÍ → Corporativo · navy/cobalt · aplica a ≥8 páginas
NO → Consultar con el equipo antes de crear un nuevo concepto

Reglas de uso por concepto

Corporativo — navy & cobalt

  • Usar en páginas institucionales sin orientación sectorial
  • Nunca usar colores de sector (verde, ámbar, noche) en estas páginas
  • Hero gradient: oscuro a navy a cobalt
  • Tipografía: Roboto Condensed 400/700 — sin excepciones
  • CTA principal: background:#0F4C81

Sectorial — color variable por sector

  • El color de concepto cambia según el sector con data-sector
  • La estructura HTML es idéntica entre los 3 sectores
  • Incluir siempre: KPI strip, 6 service cards, timeline, normas, ref-grid, 7 FAQs
  • JSON-LD obligatorio: Service + FAQPage
  • Nav marker ✓ activo en el sector correspondiente

Equipamiento — slate técnico

  • Incluir siempre tabla de especificaciones técnicas
  • Usar .norm-badge para normas IEC/IEEE/NOM
  • Acento cobre #D97706 solo para valores de spec destacados
  • JSON-LD: Product + FAQPage
  • Hero más sobrio: sin glow radial, imagen técnica preferida

Servicios Especializados — índigo

  • Incluir badge urgencia/disponibilidad 24/7
  • Checklist de beneficios antes del formulario de contacto
  • CTA flotante en mobile (posición sticky bottom)
  • JSON-LD: Service + FAQPage
  • Roboto Condensed — nunca Segoe UI en estas páginas

E-shop — slate + naranja

  • Naranja #F97316 solo para CTA de compra/cotización y precios
  • Mantener identidad VMS en logo y nav (navy)
  • JSON-LD: Product (con offers y precio)
  • Incluir ficha técnica descargable (PDF link)
  • No mezclar con tokens de otros conceptos

Snippet de implementación

Copiar en el <head> de cada página, ajustando los atributos del <html>:

<!-- 1. Atributos en la etiqueta html --> <html lang="es-MX" data-concept="sectorial" data-sector="mineria"> <!-- 2. Vincular tokens.css PRIMERO en el <head> --> <link rel="stylesheet" href="tokens.css" /> <!-- 3. Luego Google Fonts y el <style> local --> <link href="https://fonts.googleapis.com/css2?family=Roboto+Condensed:wght@400;700&display=swap" rel="stylesheet" /> <style> /* Usar var(--concept-primary), var(--concept-hero-gradient), etc. */ .hero { background: var(--concept-hero-gradient); } .kpi-strip { background: var(--concept-kpi-bg); } .kpi__val { color: var(--concept-kpi-val); } .btn--primary { background: var(--concept-btn-primary); } </style>
